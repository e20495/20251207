# 🚀 快速傳輸指南

## 步驟 1：確認前置條件

✅ **已完成**：可以 SSH 連線到樹莓派

## 步驟 2：檢查工具

在 PowerShell 中執行以下命令，檢查是否有 `scp`：

```powershell
scp
```

如果顯示使用說明，表示已安裝。如果顯示「找不到命令」，請：

### 安裝 Git for Windows（包含 scp）

1. 下載：https://git-scm.com/download/win
2. 安裝時選擇「將 Git 加入 PATH」
3. 安裝完成後**重新開啟 PowerShell**

## 步驟 3：執行傳輸

### 方法 A：使用簡化版腳本（推薦）

```powershell
.\transfer_to_pi_simple.ps1
```

### 方法 B：使用完整版腳本

```powershell
.\transfer_to_pi.ps1
```

### 方法 C：手動命令（如果腳本無法執行）

在 PowerShell 中執行：

```powershell
scp -r -P 22 ".\__2025_10_26_chihlee_pi_pico__-main" "pi@172.20.10.3:/home/pi/Documents/GitHub/2025_10_26_chihlee_pi_pico"
```

**注意**：
- 如果使用 `-P` 失敗，嘗試改用小寫 `-p`
- 會提示輸入密碼（輸入時不會顯示）
- 傳輸時間取決於資料夾大小

## 步驟 4：驗證傳輸結果

在 PowerShell 中執行：

```powershell
ssh pi@172.20.10.3 "ls -la /home/pi/Documents/GitHub/2025_10_26_chihlee_pi_pico"
```

應該會看到傳輸的檔案和資料夾。

## 🔧 常見問題

### Q1: 提示「找不到 scp 命令」

**解決方法**：
1. 安裝 Git for Windows（見步驟 2）
2. 或使用 WinSCP GUI 工具：https://winscp.net/

### Q2: 提示「Permission denied」

**解決方法**：
1. 確認密碼正確
2. 確認使用者名稱正確（通常是 `pi`）
3. 確認遠端路徑有寫入權限

### Q3: 提示「No such file or directory」

**解決方法**：
在樹莓派上先建立目錄：
```bash
ssh pi@172.20.10.3
mkdir -p /home/pi/Documents/GitHub/2025_10_26_chihlee_pi_pico
exit
```

### Q4: 傳輸速度很慢

**解決方法**：
1. 確認網路連線穩定
2. 如果使用 WiFi，嘗試改用有線網路
3. 使用 `rsync` 可以進行增量同步（只傳輸變更的檔案）

## 📝 設定檢查清單

執行前請確認：

- [ ] 樹莓派 IP：`172.20.10.3`（或您的實際 IP）
- [ ] 使用者名稱：`pi`
- [ ] 本機資料夾：`.\__2025_10_26_chihlee_pi_pico__-main`
- [ ] 遠端路徑：`/home/pi/Documents/GitHub/2025_10_26_chihlee_pi_pico`
- [ ] 已安裝 Git for Windows 或 WSL（提供 scp 命令）

## 🎯 最簡單的方法

如果上述方法都有問題，使用 **WinSCP**：

1. 下載並安裝：https://winscp.net/
2. 建立新連線：
   - 主機名稱：`172.20.10.3`
   - 使用者名稱：`pi`
   - 密碼：您的樹莓派密碼
3. 連線後，直接拖放資料夾即可
